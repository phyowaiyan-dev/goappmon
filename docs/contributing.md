# Contributing Guide

## Welcome

Contributions are welcome and appreciated.

## Good Contribution Types

- bug fixes
- tests
- documentation improvements
- architecture cleanup
- reliability and performance improvements

## Contribution Process

1. Fork or branch from the repository.
2. Make a focused change.
3. Add or update tests.
4. Update docs if behavior changes.
5. Run formatting and verification locally.
6. Open a pull request with context and screenshots or logs when relevant.

## Review Expectations

- keep pull requests small when possible
- describe what changed and why
- include follow-up work separately if it is large
- respond to review feedback clearly and respectfully

## Code Style

- follow standard Go formatting
- keep packages cohesive
- prefer readable code over clever code
